# Contributing to CyberForge

First off, thank you for your interest in contributing to **CyberForge**! 🎉

Whether you're fixing a typo, improving documentation, reporting a bug, or developing a new cybersecurity tool, every contribution helps make CyberForge a better resource for the community.

---

# Before You Start

Please take a moment to:

* Read the README.
* Review the Code of Conduct.
* Check existing Issues to avoid duplicate work.
* Search for existing Pull Requests before opening a new one.

---

# Ways You Can Contribute

There are many ways to help improve CyberForge.

## Documentation

* Improve existing documentation.
* Fix grammar or spelling mistakes.
* Create tutorials.
* Expand learning resources.

---

## Code

* Add new cybersecurity tools.
* Improve existing projects.
* Fix bugs.
* Improve performance.
* Write unit tests.

---

## Datasets

Help expand our collection of educational datasets by contributing:

* Apache logs
* Nginx logs
* SSH logs
* Windows Event Logs
* PCAP files
* Sample reports

**Only contribute datasets that you have permission to share. Do not upload sensitive, confidential, or personal data.**

---

## Labs

Create hands-on labs covering topics such as:

* Linux
* Networking
* Python
* Blue Team
* Web Security
* Cloud Security
* Digital Forensics

---

# Contribution Process

1. Fork the repository.
2. Create a new branch.
3. Make your changes.
4. Test your changes where applicable.
5. Commit your work with a clear commit message.
6. Push your branch.
7. Open a Pull Request.

---

# Coding Standards

Please:

* Write clean, readable code.
* Follow the project's coding conventions.
* Add comments where they improve understanding.
* Include documentation for new features.
* Keep pull requests focused on a single feature or fix whenever possible.

---

# Commit Message Examples

Good examples:

```text
Add Apache log parser

Fix dashboard rendering bug

Improve installation guide

Add beginner networking lab

Update README
```

---

# Pull Request Checklist

Before submitting a Pull Request, please ensure:

* [ ] My code follows the project standards.
* [ ] I tested my changes where applicable.
* [ ] I updated documentation if needed.
* [ ] I verified that existing functionality was not broken.
* [ ] My commit messages are clear and descriptive.

---

# Reporting Bugs

When reporting a bug, please include:

* Operating system
* Programming language version (if applicable)
* Steps to reproduce
* Expected behavior
* Actual behavior
* Screenshots or logs when helpful

---

# Feature Requests

Feature requests are welcome.

When submitting a request, please explain:

* The problem you're trying to solve.
* Your proposed solution.
* Why it would benefit CyberForge users.

---

# Questions

If you're unsure where to start, open a Discussion or Issue. We'd rather answer a question than have someone hesitate to contribute.
